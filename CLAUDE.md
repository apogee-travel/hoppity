# CLAUDE.md

Instructions for AI agents working ON this codebase (developing, testing, reviewing — not using as a library).

## Overview

Hoppity is a contract-driven RabbitMQ topology builder for Node.js microservices, built on Rascal. Handlers and publish declarations ARE the topology — `deriveTopology` generates all Rascal config automatically from contract declarations. Middleware remains for cross-cutting concerns only. The entry point is `hoppity.service(name, config).use(middleware).build()`, which returns a fully-wired `ServiceBroker`.

## Coding Conventions

- **TypeScript strict mode** across all packages
- **Prettier**: 4-space indent, 100-char width, double quotes, semicolons, trailing commas (es5), LF
- **ESLint**: `@typescript-eslint/no-explicit-any` is warn (not error); `_`-prefixed unused vars allowed
- **Path alias**: `@apogeelabs/*` → `packages/*/src` (root tsconfig)
- **File structure**: `src/index.ts` barrel, `src/*.ts` source, `src/*.test.ts` colocated tests
- **Build output**: `dist/index.js` (CJS), `dist/index.mjs` (ESM), `dist/index.d.ts` (types)
- **Test conventions**: See `.ai/UnitTestGeneration.md` — 3 critical rules:
    1. Execute method under test in `beforeEach`, assert in `it` blocks
    2. Configure mock behavior in `beforeEach`/`beforeAll`, not at module level
    3. Invoke callbacks via `mockImplementation`, not `mock.calls[N][M]()`

## Architecture

### ServiceBuilder

Seven phases when `.build()` is called:

1. **Topology derivation** — `deriveTopology()` generates all exchanges, queues, bindings, publications, and subscriptions from the handler declarations and publish contracts. No manual topology files needed.
2. **Topology merge** — `mergeTopology(rawTopology, derived)` combines any optional raw `BrokerConfig` with the derived topology. Raw config is the base; derived topology layers on top. This is the escape hatch for one-off infrastructure or gradual migration.
3. **Middleware pipeline** — Sequential. Each middleware receives the complete merged topology + shared `MiddlewareContext`. Returns modified topology + optional `onBrokerCreated` callback.
4. **Broker creation** — `BrokerAsPromised.create(finalTopology)` via Rascal.
5. **Handler wiring** — `wireHandlers()` subscribes event and command handlers to their queues. `wireRpcHandlers()` subscribes RPC responders.
6. **Outbound wiring** — `wireOutbound()` attaches `publishEvent` and `sendCommand` to the broker. `wireRpcOutbound()` sets up the reply queue subscription and attaches `request` and `cancelRequest`.
7. **Callback execution** — Sequential, fail-fast. Each middleware's `onBrokerCreated(broker)` callback runs in order against the fully-wired broker. On failure, broker is shut down before error propagates.

### Middleware Contract

Unchanged from pre-v1:

```typescript
(topology: BrokerConfig, context: MiddlewareContext) => MiddlewareResult;
```

- `context.data` — Mutable shared state (`Record<string, any>`)
- `context.logger` — Logger instance (defaults to `ConsoleLogger`)
- `context.middlewareNames` — Tracks execution order
- `context.serviceName` — The service name passed to `hoppity.service()`
- Return `{ topology, onBrokerCreated? }` — topology changes + optional lifecycle hook

### Key Internals

- `fast-deep-equal` detects whether middleware modified topology (for logging)
- Errors are wrapped with pipeline context (middleware name/index)
- Callback failures trigger `broker.shutdown()` before re-throwing
- `instanceId` is auto-generated (UUID) if not provided in `ServiceConfig`
- RPC reply queue is per-instance (`{serviceName}_{instanceId}_reply`), exclusive + auto-delete
- Broker `shutdown()` is wrapped by RPC wiring to drain pending requests before shutdown

## Things to Watch Out For

- **Test style is strictly enforced** — `.ai/UnitTestGeneration.md` has 3 critical rules. Violations will fail code review. Read it before writing any tests.
- **Pre-commit hooks run Prettier** — Formatting is automatic. Don't fight it.
- **turbo caching** — `test` depends on `build`. If you change source, `pnpm build` before `pnpm test`, or use `pnpm dev` for watch mode.
- **Integration tests need Docker** — Testcontainers spins up RabbitMQ. Tests run serially (`maxWorkers: 1`), 60s timeout.
- **Logger goes in config, not middleware** — Pass `logger` in `ServiceConfig` directly. It's active before any middleware runs, so there's no ordering concern.
- **Middleware is synchronous** — The middleware function itself returns `MiddlewareResult` (not a Promise). Async work goes in `onBrokerCreated`.
- **No deep imports** — Every package exports only from its root. No `dist/` or sub-module imports exist.
- **Node 22 required** — `.nvmrc` specifies it. `structuredClone` and other modern APIs are assumed available.
- **No topology files** — The v1 API derives topology from handler declarations. Don't write manual `topology.ts` files. Use the `topology` escape hatch in `ServiceConfig` only for infrastructure that can't be derived (e.g., DLX exchanges, shovel config).
- **No `withTopology()` or `RascalBuilder`** — Those are gone. The one path is `hoppity.service()`. The escape hatch is `service("name", { topology: rawConfig })` with no handlers/publishes.
- **No `withDelayedPublish` middleware** — Delayed delivery is built into core. Declare `delay: true` or `delay: { default: N }` on contracts in `defineDomain`, then pass `{ delay: ... }` to `publishEvent`/`sendCommand`. No middleware needed.
- **Delayed delivery is per-handler, not per-service** — Each consuming service gets its own wait/ready/error queue triple. The queues are named `{service}_{domain}_{opType}_{snake_name}_{wait|ready|errors}`.
