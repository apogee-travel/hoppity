Instructions for AI agents working ON this codebase (developing, testing, reviewing вҖ” not using as a library).

## Overview

Hoppity is a pattern-driven RabbitMQ topology builder for Node.js microservices, built on Rascal. It provides a composable middleware pipeline where each middleware receives a `BrokerConfig` + shared `MiddlewareContext`, can modify both, and optionally return an `onBrokerCreated` callback. The builder pattern (`withTopology().use(middleware).build()`) produces a configured Rascal `BrokerAsPromised`.

## Coding Conventions

- **TypeScript strict mode** across all packages
- **Prettier**: 4-space indent, 100-char width, double quotes, semicolons, trailing commas (es5), LF
- **ESLint**: `@typescript-eslint/no-explicit-any` is warn (not error); `_`-prefixed unused vars allowed
- **Path alias**: `@apogeelabs/*` вҶ’ `packages/*/src` (root tsconfig)
- **File structure**: `src/index.ts` barrel, `src/*.ts` source, `src/*.test.ts` colocated tests
- **Build output**: `dist/index.js` (CJS), `dist/index.mjs` (ESM), `dist/index.d.ts` (types)
- **Test conventions**: See `.ai/UnitTestGeneration.md` вҖ” 3 critical rules:
    1. Execute method under test in `beforeEach`, assert in `it` blocks
    2. Configure mock behavior in `beforeEach`/`beforeAll`, not at module level
    3. Invoke callbacks via `mockImplementation`, not `mock.calls[N][M]()`

## Architecture

### Core Pipeline (RascalBuilder)

Three phases when `.build()` is called:

1. **Middleware execution** вҖ” Sequential. Each middleware receives current `BrokerConfig` + shared `MiddlewareContext`. Returns modified topology + optional `onBrokerCreated` callback.
2. **Broker creation** вҖ” `BrokerAsPromised.create(finalTopology)` via Rascal.
3. **Callback execution** вҖ” Sequential, fail-fast. Each `onBrokerCreated(broker)` runs in order. On failure, broker is shut down before error propagates.

### Middleware Contract

```typescript
(topology: BrokerConfig, context: MiddlewareContext) => MiddlewareResult;
```

- `context.data` вҖ” Mutable shared state (`Record<string, any>`)
- `context.logger` вҖ” Logger instance (defaults to `ConsoleLogger`)
- `context.middlewareNames` вҖ” Tracks execution order
- Return `{ topology, onBrokerCreated? }` вҖ” topology changes + optional lifecycle hook

### Key Internals

- Topology is `structuredClone()`d at builder construction (prevents external mutation)
- `fast-deep-equal` detects whether middleware modified topology (for logging)
- Errors are wrapped with pipeline context (middleware name/index, execution count)
- Callback failures trigger `broker.shutdown()` before re-throwing

## Things to Watch Out For

- **Test style is strictly enforced** вҖ” `.ai/UnitTestGeneration.md` has 3 critical rules. Violations will fail code review. Read it before writing any tests.
- **Pre-commit hooks run Prettier** вҖ” Formatting is automatic. Don't fight it.
- **turbo caching** вҖ” `test` depends on `build`. If you change source, `pnpm build` before `pnpm test`, or use `pnpm dev` for watch mode.
- **Integration tests need Docker** вҖ” Testcontainers spins up RabbitMQ. Tests run serially (`maxWorkers: 1`), 60s timeout.
- **structuredClone** вҖ” Topology is deep-cloned at builder construction. Don't rely on reference equality between input and what middleware receives.
- **Middleware order matters** вҖ” `withCustomLogger` should go first so downstream middleware uses the custom logger. Subscriptions should go last (after topology is finalized).
- **Node 22 required** вҖ” `.nvmrc` specifies it. `structuredClone` and other modern APIs are assumed available.
- **Middleware is synchronous** вҖ” The middleware function itself returns `MiddlewareResult` (not a Promise). Async work goes in `onBrokerCreated`.
- **No deep imports** вҖ” Every package exports only from its root. No `dist/` or sub-module imports exist.
